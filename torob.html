    Torob Redesign

    Company: Torob Products: Torob Web app, Android app and Website My role: UX Designer, UI Designer, Product Strategist, User Researcher Team: 4 Engineers, myself 2019 - 2020

    Overview

    Torob allow users to find best price for products in online stores. User can compare prices and get notifications on price changes. As senior product designer, I was responsible for redesigning overall UI, revisioning product features and improving UX. {fig. desktop view, before and after design} {fig. mobile view, before and after design}

    Our redesign process

    We tune design thinking process based on our requirements. {fig}

    Finding and understanding Problems

    For understanding current problems we should consider many aspects of product. So we do these:

    • Analyzing current data and gathering new data
    • Considering user feedbacks for app reviews, social networks and support teams
    • Doing surveys
    • Doing Usability tests
    • Reconsidering persona
    • Interview with other teams

      Prioritize problems

    For prioritizing problems, We interview with stakeholders and managers of all teams to have a realistic estimation of time and resources and balance problems and OKRs.

    Ideate

    • Brainstorm session
    • Considering competitors

      prototypes

    For critical features or radical changes, We completely implement them in a specific version of website or app to do necessary tests and gathering data. But for Ideate and prioritize problems sessions, We use low-fi prototype which craft them by hand drawing or using application like Sketch, Invision and Principle.

    Tests

    We found any change in main parts of web or app, can effect directly on revenue so A/B tests are an inseparable part of our design process. We also do usability tests by user interviews and watching live interaction of users to improve UX and tests new features.

    UI redesign

    At first, We studied typography and colors as basic fundamentals for UI components. {Typography img} {Color img}

    After that, We designed basic interface components and made a UI kit as a library in Sketch.

    The UI kit help us to build main components and pages more quickly and consistently. If new components was needed, We first designed it in UI kit to ensure consistency with other components, then add it to main design.

    We should design UI components for mobile and desktop devices (Web and Android) and design must be completely responsive. Therefor, We should design some components and pages in different size of screens.

    For critical pages, different versions was designed to cover all devices. For example, We designed 9 version of Product Page which is most visited page of Torob.

    But there was some interesting and difficult challenges in redesigning process.

    We don’t Want design a custom version for every platform and device sizes so we should design components in a way that works on all platform as much as possible. Therefor, We consider a little bigger dimensions for some components like checkbox, radio button, input fields to work on all device sizes.

    Case study of UX problems

    Ranking store: From lowest price to best price

    Torob sorted stores based lowest price. Price is the most important factors in shopping experience but is not the only one. Most shopper care about delivery time, customer feedback, guarantee and other factors. So we decided to change sorting of stores to improve our customer experiences.

    Case study of UI Design Challenges

    Style guide

    To be ensure consistency over all designs, we need a style guide. I worked with PMs and developers to craft a style guide for Torob, which contains the library of common components used in the product and advise on usage and implementation. It described from typography to entire screens structure. We designed fundamental UI elements and progressively craft more complex components.

    Keeping it simple

    Services like Torob, commonly have very messy interface. they have a lot of features and information and directly put them on interface. Keeping interface simple and beautiful beside a lot of features was a serious challenge. The home page of Torob show how much we care about simplicity.

    Red was everywhere

    Torob’s brand color is red. Choosing red as accent color and put it every where, make it scary for user. It looks there is a lot of alarms and errors on screen. For example look at this page, Which button is CTA button? Confusing! {fig} And there was more than three different red colors in UI elements. After many iterations, choosing a less saturated color, removing unnecessary usage and using it more precisely, We craft a new design. But Torob revenue is directly depeond on “BUY” buttons clicks and users used to the current design. So we must choose visual styles with great caution and did a/b tests to find effects of new design on user clicks. {fig}

    Search redesign

    Company: Torob Products: Android app My role: Sole Designer, UX and UI Designer, User Researcher and Product Strategist Team: 4 Engineers, 1 product manager and myself Design approach: Data Driven 2019 - 2020

    Overview

    Torob target is to be a shopping search engine and try to be more than just a price comparison service .So search is the hearth of Torob. If you look at Torob.com, home page is very similar to Google.com to show main target of Torob. But design of Android app is different. It is not search centered.

    Challenge

    Understanding current experience of search How we can improve search experience for users? How put search in the center of user experience? How motivate users to a more accurate search?

    Understanding current experience of search

    To understand current experience of users we use Amplitude to measure specific events and

    How we can improve search experience for users?

    Fortunately, there is a lot of app with search feature and great researches about search. We consider search experience in price comparison website, major e-commerce website and well-known app to find tested patterns and good experiences.

    After considering different ideas, We choose a design for test. We release a version with new design for a limited group of users by Google Play and using Amplitude to measure events. This is our common approach for big features. To consider impact of new design, we defined two events:

    1. Total visit of product page (PP)
    2. Total click on buy button (BB)

    And defined Search Type for different ways of search. Direct: User hit enter key after typing in search field and don’t choose any suggestions. Suggestion: Click on suggestion row after typing in search field. There is two type for suggestions; by text and categories. History: Click on a product card which seen previously by user. Top product: Click on most visited product card in previous day. Top query: Click on most search query in previous day. Voice: Number of search by voice command.

    Top Query and Top Product During our research, We found Torob app don’t show any reaction to product trend and social events. For example at reales of PS5, user couldn’t see any trace of it in app only by searching it or before 4 season changes in Iran, there is a lot of trend and on sale events in shopping that ignored in app. We decided to use users search query and product visit to detect these trend and show that on search section. We hope this feature motivate user to search more and increase retention of new user to find trend.
